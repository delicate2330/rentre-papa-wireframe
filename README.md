# 파파 그룹핑 (OPD-79) — 수신함 / 구성원 와이어프레임

세일즈포스팀 파트너 파일럿의 **퇴사자 프로세스 / 미답변 관리 / 구성원 그룹핑**을 다루는 인터랙티브 와이어프레임입니다.

🔗 **데모:** https://delicate2330.github.io/rentre-papa-wireframe/

> 단일 HTML 파일이며, 모든 동작은 더미 데이터로 시뮬레이션됩니다.

## 화면 구성

좌측 아이콘 사이드바에서 **수신함**과 **구성원** 두 뷰를 전환합니다.

### 1. 수신함
- **내 담당**: 내 상담 / 신규 인입 메시지(레드닷) / 미답변(연한 붉은 방). 클릭 시 상담 리스트 필터.
- **팀 전체**: 전체 / 신규 인입 메시지 / 미답변 — 각각 채팅·전화(TM)로 분기. 카운트는 실제 방 데이터와 연동.
- **구성원 그룹핑**: 재직상태 → 역할(관리자/컨설턴트) → 상담수단(채팅/TM) → 카테고리(제품) 4단계 중첩. 미답변 건수 표시, 카테고리 접기/펼치기, 상담원 검색.
- **상담 리스트**: 진행중/종료, 채널 서브탭, 상태 태그(상담중·주문확정·설치대기·지원금대기·계약완료, 건수 표시), 이름/전화번호 검색. 미답변 방은 항상 최상단.

### 2. 구성원
- 재직/퇴사 리스트 + 4단계 필터 + 검색 + 페이지네이션(10/20/100명).
- 담당 채팅방·미답변 건수 컬럼.
- 행 클릭 → 상세 패널(담당 방, 이관/수신 이력).
- **퇴사 처리** 시 담당 채팅방 재배분: 단일 이관 / 균등 분배 / 가중 분배(부하↓ 우선). 균등·가중은 **퇴사자와 같은 카테고리 내**에서만, 누구에게 몇 건 가는지 미리보기 표시. 실행취소 지원.

## 미답변 흐름 (데모)

1. **신규 인입** 방에 답장 전송 → 표시 사라지고 **내 상담**으로 편입
2. 채팅방 상단 **🔔 고객 답변(데모)** 버튼 → 고객 메시지 추가 + **미답변** 전환
3. 답장 다시 전송 → 미답변 해제

> 미답변은 "안 읽음"이 아니라 **답변 여부** 기준입니다.
